Marco teórico Definición e historia de cloud computing La computación en la nube o cloud computing corresponde a una estrategia que permite ofrecer servicios, tales como servidores, almacenamiento de datos, etc. a través de internet, por lo que representa una opción para pequeñas, medianas y grandes empresas que requieran el uso de alguno de estos servicios y su accesibilidad continua. Está idea fue implementada (en una proporcion considerablemente grande) alrededor de 2006, principalmente por empresas de carácter global, tales como Google (Google Cloud Services), Amazon Web Services y Microsoft (Microsoft Azure), sin embargo el fundamento de la computación en la nube es un pensamiento que se atribuye a informáticos como Joseph Carl Robnett Licklider o John McCarthy, quienes, alrededor de 1960, fueron quienes iniciaron con esta corriente basados en el hecho de hacer que la web fuera un medio para trasmitir programas y datos a nivel global. Pero, aunque empresas como Google, Amazon y Microsoft fueron quienes introdujeron el cloud computing a mayor escala, no fue ninguno de ellos quien logró los primeros avances en este ámbito, sino una empresa estadounidense llamada Salesforce, quien en 1999 logró el desarrollo de una página web que ofrece aplicaciones empresariales a través de internet. Tras este hecho muchas otras empresas comenzaron a introducirse a la corriente del cloud computing, hasta llegar a la realidad actual de este tipo de computacion. El cloud computing ofrece diferentes modelos de servicio para satisfacer las necesidades de las empresas como lo son SaaS, PaaS y IaaS. Conceptos básicos Computación: esta es la ciencia que se encarga de entender, mantener y desarrollar equipos computacionales como también de sus equipos periféricos. También se encarga del funcionamiento de estos mediante operaciones matemáticas que son la base de toda ciencia. Nube: este es un concepto abstracto ya que se refiere a la conexión de múltiples servidores conectados mediante la red que funcionan para almacenar y administrar datos. Esto para facilitar el acceso a estos datos. En palabras más simples se podría decir que metafóricamente es como usar un almacenamiento en línea, en vez de usar el del computador. Internet: es una red de conexiones entre computadores la cual permite la fácil transferencia de información sin necesidad de que estas estén conectadas entre sí. Servicios web: este concepto está relacionado con el de internet, y se refiere al intercambio de información entre dos dispositivos mediante un puerto específico, sirven para resolver problemas muy específicos como por ejemplo en una aplicación, un dispositivo hace una solicitud de información, el otro la recibe y responde enviando dicha información. Funciona con archivos como Json y HTML. Explicación de los modelos de despliegue Pública: En los modelos públicos los servicios son ofrecidos por empresas o proveedores que se encargan del mantenimiento y seguridad de los servicios ofrecidos. En este caso los usuarios no tienen necesidad de adquirir hardware, software o equipo de soporte, ya que este es gestionado por la empresa o proveedor. Privada: Los modelos privados se caracterizan por ser utilizados por una sola organización, aunque pueden ser gestionados por terceros y pueden estar alojados tanto interna como externamente en la organización. Además este tipo de modelos se caracterizan por no poseer puntos de acceso público Híbrida: Se componen de dos infraestructuras distintas, ya que poseen una base del modelo privado pero aplican funcionalidades y características de un modelo público.} Explicación de los modelos de servicio IaaS: El IaaS (Infrastructure as a Service) se caracteriza por proveer recursos informáticos, como servidores, redes, almacenamiento y espacio en centro de datos a las empresas que utilizan su un servicio con este modelo. SaaS: Este modelo conocido como SaaS (Software as a Service) es trabajado en línea y no por la empresa. Esto permite que la empresa no tenga que preocuparse sobre mantenimiento de software, manejo de estructura, seguridad de red y demás servicios relacionados con mantener las aplicaciones en correcto funcionamiento. En el modelo SaaS se procura hacer más fácil y cómodo el acceso a las aplicaciones y servicios ofrecidos a través de una interfaz de navegador o de un programa. PaaS: PaaS (Platform as a Service) es como una combinación entre IaaS y SaaS, está caracterizado por garantizar un entorno que provea todos los servicios y requisitos necesarios para dar soporte a la creación y desarrollo de aplicaciones basadas en la web sin la necesidad de instalar IDEs ya que todo se desarrolla en línea. Comparativa de tres principales proveedores de Cloud Computing con respecto a su oferta general, posicionamiento en el mercado y beneficios para los developers: Amazon Web Services: es una colección de servicios de computación en la nube pública (también llamados servicios web) que en conjunto forman una plataforma de computación en la nube, ofrecida por Amazon.com, es usado en aplicaciones populares como Dropbox, Foursquare, HootSuite. Es una de las ofertas internacionales más importantes de la computación en la nube y compite directamente contra servicios como Microsoft Azure y Google Cloud Platform. Microsoft Azure: es un servicio de computación en la nube creado por Microsoft para construir, probar, desplegar y administrar aplicaciones y servicios mediante el uso de sus centros de datos. Proporciona software como servicio (SaaS), plataforma como servicio (PaaS) e infraestructura como servicio (IaaS) y es compatible con muchos lenguajes, herramientas y marcos de programación diferentes, incluidos software y sistemas específicos de Microsoft y de terceros. Google Cloud: es una plataforma donde se encuentran las aplicaciones de desarrollo web que google ofrece. Esta nube ofrece servicios como lo son almacenamiento y análisis de datos, machine learning, etc. Además esta provee los tres tipos de modelos de servicios (SaaS, PaaS, IaaS) Análisis de resultados: A partir de los datos analizados es posible responder a las preguntas inicialmente planteadas, ya que, por una parte, se esclareció el concepto y la explicación del cloud computing, desde sus inicios, hasta lo que se conoce y se ha logrado desarrollar actualmente. En cuanto a recomendaciones sobre el cloud computing, esta es una herramienta muy útil que con el paso del tiempo irá creciendo más, aumentando sus utilidades y también su eficiencia. Por lo que no es de extrañar que las empresas quieran utilizarla, pero ¿Qué modelo utilizar? Bueno esto depende de muchos factores, pero los que pueden ser de mayor peso son el tamaño de la empresa como los gastos que esto implica para la empresa. Con esto en mente ya se puede decidir qué modelo utilizar, en el caso de los de despliegue, digamos que es una empresa muy grande, entonces podría usar el modelo privado ya que este traerá ventajas como no tener acceso público, y para el caso de los modelos de servicio dependen de lo que la empresa necesite, digamos que la empresa empezará con un nuevo proyecto pero no posee los recursos necesarios para ello, aquí el modelo más conveniente será el IaaS ya que este proveerá estos recursos a la empresa.
